# @feanorfs/agent

Native Node.js bindings for the FeanorFS agent SDK (`spawn` / `status` / `refresh` / `land` / `clean` / `conflicts keep`).

**Status:** release-ready, not yet claimed as published. Trusted-tag CI builds and
verifies five native packages before publishing the facade. Build from source or
install local tarballs until the first npm registry release completes.

Supported packages:

- `@feanorfs/agent-darwin-x64`
- `@feanorfs/agent-darwin-arm64`
- `@feanorfs/agent-linux-x64-gnu`
- `@feanorfs/agent-linux-arm64-gnu`
- `@feanorfs/agent-win32-x64-msvc`

## Build

Requires Rust (same toolchain as the repo) and Node 18+:

```bash
# from repo root
cargo build -p feanorfs-client   # optional: for integration test workspace setup

cd bindings/ts
npm ci
npm run build                    # compiles feanorfs-agent-node.node via napi-rs
npm run create-npm-dir           # generates exactly five platform manifests
npm run assemble-packages        # collects and hashes all five CI artifacts
npm run verify-packages          # read-only metadata/artifact verification
```

The native module is written to `feanorfs-agent-node.<platform>.node` in this directory. `index.js` (generated by napi-rs) loads it.

## Usage

Typed async API (recommended):

```javascript
import { spawn, land, clean } from '@feanorfs/agent'

const result = await spawn('/path/to/workspace', 'worker', { noSync: false })
await land('/path/to/workspace', 'worker')
await clean('/path/to/workspace', 'worker')
```

Low-level JSON strings:

```javascript
import { native } from '@feanorfs/agent'
const json = await native.agentSpawn(root, name, {})
```

Each call opens the workspace fresh (`local_state.json` + transport). Pass the workspace root path every time.

## Contract

JSON shapes match [docs/agent-api.md](../docs/agent-api.md). TypeScript interfaces are in `contract.d.ts`.

## Test

```bash
FEANORFS_BIN=/path/to/feanorfs npm test
```

Uses the CLI only to create a local-hub workspace; the loop itself runs through the native module.

Release assembly also runs `npm pack --dry-run --json` for all six packages,
installs the facade plus host tarball with the registry disabled, and executes
the same loop. `scripts/publish-packages.mjs` makes retries idempotent by
matching registry tarball integrity and metadata before any publish.

# @feanorfs/agent

Native Node.js bindings for the FeanorFS agent SDK (`spawn` / `status` / `refresh` / `land` / `clean` / `conflicts keep`).

**Status:** private — build from source until prebuilt binaries ship from CI.

## Build

Requires Rust (same toolchain as the repo) and Node 18+:

```bash
# from repo root
cargo build -p feanorfs-client   # optional: for integration test workspace setup

cd bindings/ts
npm ci
npm run build                    # compiles feanorfs-agent-node.node via napi-rs
```

The native module is written to `feanorfs-agent-node.<platform>.node` in this directory. `index.js` (generated by napi-rs) loads it.

## Usage

Typed async API (recommended):

```javascript
import { spawn, land, clean } from '@feanorfs/agent'

const result = await spawn('/path/to/workspace', 'worker', { noSync: false })
await land('/path/to/workspace', 'worker')
await clean('/path/to/workspace', 'worker')
```

Low-level JSON strings:

```javascript
import { native } from '@feanorfs/agent'
const json = await native.agentSpawn(root, name, {})
```

Each call opens the workspace fresh (new cache DB + transport). Pass the workspace root path every time.

## Contract

JSON shapes match [docs/agent-api.md](../docs/agent-api.md). TypeScript interfaces are in `contract.d.ts`.

## Test

```bash
FEANORFS_BIN=/path/to/feanorfs npm test
```

Uses the CLI only to create a local-hub workspace; the loop itself runs through the native module.
